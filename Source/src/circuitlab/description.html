A benchmark for Petit Poucet

Petit Poucet is Java library that allows users to create function circuits over which lineage queries can then be addressed.

This benchmark measures the impact on performance (processing time and memory) of the lineage features. It does so by evaluating various function circuits on multiple types of inputs, and comparing time and memory with lineage being toggled on or off. Note that for historical reasons, this benchmark uses the branch 1.x of Petit Poucet; since the benchmark was created, a new version 2.x has been implemented; it provides similar features but with a completely different API.

Set of problems

The problems correspond to simple-to-medium load computing tasks.

  • Window average: given a CSV file, this task extracts the numerical value in each line, compute the average of each set of n successive values and check that it is below some threshold t
  • Iterator: evaluates a temporal logic expression over a sequence o method calls. The property stipulates that on a Java \texttt{Iterator}, every call to method next must be followed by a call to hasNext. A function circuit returning the false verdict can be explained by retracing the locations in the sequence where two successive next events can be found.
  • Count regex matches: calculates the number of matches of a given regular expression withing a character string. The explanation for the output number retraces the character ranges of each sub-string matching the pattern in question.
  • Sum of triangle areas: given a list of arbitrary vectors, this task checks that each vector contains the lengths of the three sides of a valid triangle; if so, it computes their area using Heron’s formula 6 and sums the area of all valid triangles.
  • Nested bounding boxes: given a DOM tree, this task checks that each element has a bounding box (width and height) larger than all of its children.
  • Get all numbers: represents a simple operation that takes an input comma- separated list of elements, and produces a vector containing only the elements of the file that are numerical values. The explanation we ask is to point at a given element of the output vector, and retrace the location in the input string that corresponds to this number.

Each of these problems comes in configurable "sizes"; for example, Window average varies the number of elements in the input file, while Nested bounding boxes varies the number of nodes in the DOM tree.

References

This is version 2 of the benchmark, which extends version 1 with additional tasks and measurements. The results of version 1 were mentioned in the following publication:

  • S. Hallé, H. Tremblay. (2021). Foundations of Fine-Grained Explainability. Proceedings of the 33rd International Conference on Computer-Aided Verification (CAV 2021) -- Part II. Springer: Lecture Notes in Computer Science 12760, 500-523.

Prerequisites

The self-contained JAR file is a bundle of the following dependent libraries (which you don't need to install):

  • Azrael
  • LabPal
  • Petit Poucet

How to run this lab

This lab can be run by typing, at the command line:

java -jar petitpoucet-circuit-lab-v2.jar --autostart

See the Help page for more information.
